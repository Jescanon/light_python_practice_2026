# Индексатор (полный вариант)

Консольная утилита, которая обходит папку и складывает данные в SQLite.

Группа: ПИЭ-41
ФИО: Смараков Александр Владимирович
Вариант: полный

## Что где лежит
- main.py - запуск, разбор аргументов
- config.py - пути к базе и логам
- database/db.py - sqlite, создание таблиц, соединение через контекстный менеджер
- logs_service/logs.py - логи (-v это INFO в консоль, -vv это DEBUG, в файл всегда полный)
- service/scaner.py - сканер: обход папки и запись индекса (+ хэши файлов)
- service/reports.py - показ пользователю: list, history, duplicates, checks
- service/hasher.py - считает хэш содержимого файла (blake2b, читает кусками)
- service/compare.py - сравнение папки с бeкапом (compare)
- tests/ - тесты на сканер, хэшер, отчёты и сравнение

База data/app.db и лог logs/indexer.log создаются сами при запуске.

## Установка

Чтобы вызывать как indexer, а не через python -m, нужно один раз поставить пакет. Лучше в venv:

```
python -m venv venv
venv\Scripts\activate
pip install -e .
```

-e (editable) - ставит ссылку на код, а не копию, так что правишь файлы и не надо
переустанавливать. После этого появляется команда indexer (pip берёт её из pyproject.toml,
секция [project.scripts]). Без установки команды нет - голое клонирование её не создаёт,
её генерит именно pip при установке.

После установки запуск короче:

```
indexer scan <папка>
indexer compare <источник> <бэкап>
```

Удалить: pip uninstall indexer.

## Запуск

Можно и без установки, из корня проекта через -m (просто python src/main.py не работает,
ругается на импорты):

```
python -m src.main scan <папка>
```

Дальше в примерах пишу python -m src.main, но если поставил пакет (см. Установка) - просто
заменяй это на indexer, остальное один в один.

Флаги -v, -vv и --db идут ДО слова scan, а не после:

```
python -m src.main -v scan C:\папка
python -m src.main -vv scan C:\папка
python -m src.main --db C:\tmp\my.db scan C:\папка
```

-v показывает INFO в консоль, -vv ещё и DEBUG (что нашёл, что хэширую и пр.). В файл лог пишется
всегда целиком, флаги влияют только на вывод в консоль, сделано было для упрощенного дебага.

Фильтры у scan:
- --ext .py,.txt - по расширению
- --name отчет - по куску имени

-v это подробный лог, --db можно не писать, тогда база в data/app.db.

## list - показать индекс
Печатает, что уже лежит в базе по этой папке (без нового сканирования):

```
python -m src.main list <папка>
```
Выводит размер и путь по каждому файлу, в конце общее количество. Если пусто - подскажет
сначала сделать scan. Папку указывай так же, как при scan.

## history - история запусков
Показывает прошлые сканы этой папки (без нового сканирования):

```
python -m src.main history <папка>
```
По каждому запуску печатает когда сканили и сколько найдено / добавлено / изменено / удалено,
от свежих к старым. Берётся из таблицы scans.

## duplicates - поиск дубликатов
Показывает файлы с одинаковым содержимым (по хэшу), без нового сканирования:

```
python -m src.main duplicates <папка>
```
Группирует файлы по хэшу и печатает группы, где файлов два и больше. Имя/расширение/папка
не важны - сравнивается именно содержимое, так что копию в другой папке тоже видно. Если
дубликатов нет (или базу ещё не сканили) - так и скажет. Хэши берутся из таблицы files,
их считает scan.

## compare - сравнить папку с бэкапом
Сверяет исходную папку с её резервной копией и показывает различия:

```
python -m src.main compare <источник> <бэкап>
```
Бэкап - это вторая реальная папка (копия, которую ты сам куда-то сложил), программа её НЕ
создаёт, только читает. Внутри сканит обе папки в индекс и сравнивает по содержимому (хэшу),
а не по дате - копия всегда скопирована позже, по mtime было бы "всё изменено". Показывает три
группы: чего нет в бэкапе, что изменилось, что в бэкапе лишнее. Если всё совпало - так и скажет.

## checks - история проверок
Показывает прошлые сравнения по этому источнику (без новой проверки):

```
python -m src.main checks <источник>
```
По каждой проверке печатает когда сверяли и сколько нет / изменено / лишних, от свежих к старым.
Берётся из таблицы checks.

## Что делает scan
- рекурсивно обходит папку (iterdir + рекурсия руками, без rglob), пропускает venv/.git/.idea/__pycache__
- по каждому файлу берёт относительный путь, размер, дату изменения, расширение
- считает хэш содержимого (blake2b) и кладёт в базу
- сохраняет в таблицу files - новые добавляет, изменённые обновляет
- если файл не менялся (size и mtime те же и хэш уже есть) - заново не хэширует, мини оптимизация)))
- файлы, которых больше нет на диске, убирает из индекса
- пишет строку в таблицу scans (история запуска) и в лог - сколько добавлено / изменено / удалено

## Тесты
Лежат в tests/, обычный unittest (без сторонних либ). Запуск из корня:

```
python -m unittest discover -s tests -t .
```
